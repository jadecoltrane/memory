# 坑

记忆库仓库 `jadecoltrane/memory` 从建库起一直是 GitHub **公开仓库**,健康、心理自我工作、生活偏好、工作台问答等隐私内容对任何人可见,直到 2026-07-04 做系统体检时才发现。

# 原因

建仓库时没有检查 visibility。整套规则(CLAUDE.md)写满了防编造、防茧房、人工否决权,却没有一条提到隐私边界——盲区不在执行层,在规则本身:没人想到要问"这些内容给谁看"。

# 解法

- 2026-07-04 已执行 `gh repo edit --visibility private` 转私有
- CLAUDE.md 开头新增「隐私边界」规则:仓库必须保持 private,AI 不得转公开、不得把记忆内容发布到外部服务

# 残留风险(何时失效)

公开期间的内容可能已被搜索引擎缓存、爬虫抓取或他人 fork,转私有只能止损、无法追回。若未来发现某条记忆的暴露有实际影响,按具体内容单独评估。本条长期有效,除非仓库迁移到其他托管方式。
