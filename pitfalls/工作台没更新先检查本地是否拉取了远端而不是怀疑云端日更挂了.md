# 工作台没更新先检查本地是否拉取了远端,而不是怀疑云端日更挂了

**现象**:2026-07-06 电脑和手机上的工作台都停在前一天,看起来像每日生成挂了。

**真实原因**:云端 GitHub Actions「工作台日更」当天早上 05:57(北京时间)已成功跑完并推送,但本地 iCloud 库没有从 GitHub 拉取。电脑和手机共用同一份 iCloud vault,所以只要本地 clone 落后,两个设备一起显示旧版。Obsidian Git 插件虽配置了 autoPullOnBoot + 每 10 分钟自动拉取,但只在 Obsidian 打开且插件正常运行时生效——当天 Obsidian 没打开过(当天没有任何 vault backup 提交可佐证)。

**排查顺序**(下次遇到"工作台没变化"):
1. 看本地 `工作台.md` 的 `generated:` 日期
2. `gh run list` 查云端「工作台日更」workflow 是否成功——成功就是本地没 pull,`git pull` 即可解决;失败才是生成侧的问题
3. 区分信号:横幅心跳报警(generated 超 48h 显红)反映的是**本地这份文件**的日期,本地不 pull 时它会误报"系统挂了",此时云端可能一切正常

**何时失效**:如果以后工作台生成方式改回本地定时任务,或同步机制不再依赖 Obsidian Git 拉取,此坑不再适用。相关:本地定时任务 daily-workbench-refresh 已停用,由云端 .github/workflows/daily-workbench.yml 接管。
