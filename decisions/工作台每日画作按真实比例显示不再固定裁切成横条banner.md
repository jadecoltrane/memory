# 决定

工作台「🖼 每日画作」不再用固定宽高比的横条 banner(`background-image` + `background-size: cover` 裁切),改成真实 `<img>` 按画作本身比例显示;画名+简介移到画的旁边(宽屏)或下方(窄屏/手机),不再压在画上。

# 为什么

调研了几幅常入选这里的名画实际尺寸:戴珍珠耳环的少女 44.5×39cm(约 1.14:1)、蒙娜丽莎 77×53cm(约 1.45:1)、呐喊 91×73.5cm(约 1.24:1)、星夜 74×92cm(约 0.80:1)——绝大多数是竖长或接近正方形,只有格尔尼卡 349×776cm 这类壁画级作品才是超宽横幅。原来的横条 banner 是按"横向"预设的比例硬裁,竖画会被切掉大半构图(实测维米尔那幅脸被切掉一半),这不是个别画的问题,是布局假设和画作真实分布不匹配。

# 实现方式

宽屏(>520px):flex 横排,图占约 46% 宽(`max-height` 兜底避免极端竖长画过高),文字在右边,`min-width` 保证不会被挤没。窄屏/手机(≤520px,`matchMedia` 判断):改竖排,图在上(完整显示,`max-height:260px` 封顶)、文字在下。

刻意全部用内联样式 + `window.matchMedia` 做断点判断,不写进 `.obsidian/snippets/workbench.css`——那个文件本地专属、不入库,AI 在远程会话里看不到也改不了,写在 工作台.md 的 dataviewjs 里才能保证以后任何一次会话都能直接改这块布局,不用先问用户要本地文件内容。

原来给 banner 图做的超慢 Ken Burns 呼吸效果(放大缩小)顺带去掉了——效果依赖背景图裁切显示,和"完整显示画作比例"的新目标冲突,没有勉强保留。
