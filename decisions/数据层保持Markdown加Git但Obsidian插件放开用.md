# 决定

数据层不变:记忆的真身仍是 `jadecoltrane/memory` 仓库里的纯 Markdown + Git,GitHub 仍是 AI 与用户之间的桥梁,保证任何工具将来都能接入。变的是使用层:**Obsidian 插件放开用**(Dataview、Obsidian Git、Claudian 等),AI 生成的页面(如根目录「工作台」)可以依赖插件渲染动态内容。

# 为什么

- 2026-07-04 用户明确撤销"不绑定工具"的表述:"这个给我去了,我也可以装插件"
- 原决定真正要保护的是**数据可迁移性**,这由"内容是纯 Markdown"保证,和装不装插件无关;把两件事捆在一起是过度约束
- 插件的动态视图(如 Dataview 自动列最近输入、幽灵概念)是零 token 成本的——把机械的活交给插件、把需要判断的活留给 AI,正好符合 [[decisions/双链优先指向概念页而非笔记之间直接互链]] 里的懒加载原则

# 何时失效

某个重度依赖的插件停止维护,或内容开始以插件私有格式存储(数据可迁移性受损)时,重新评估。
