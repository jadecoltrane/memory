# 决定

AI 往记忆库(尤其 `notes/`)写笔记时,双链优先内嵌在正文中概念出现的位置,用 `[[路径|显示文字]]` 让链接自然融进句子;末尾的「相关:」列表只留给正文没提到、但值得关联的笔记,不再作为双链的唯一落点。

# 为什么

- 双链的本意是"对概念进行关联":读到哪个概念就能从哪里岔开去看,末尾列表丢失了这个上下文
- 图谱效果两种写法相同(都是一条边),差别在阅读体验;内嵌式才有"顺藤摸瓜"的感觉
- 2026-07-03 用户看到笔记只在末尾附「相关:」后明确提出想要概念级关联,经确认采纳为约定
- token 成本可忽略:一条链接只比纯文字多几个字符,且开工流程本就是按需读取而非全库读入

# 何时失效

用户表示更喜欢别的链接风格时,按「人类有否决权」直接修正。
