# 记忆库改动直接推 main,不用开 PR

**为什么**:这是单人仓库,没有协作审批需求;PR 唯一的价值是留一页改动说明,但 commit message 已经承担了这个职责,多一道 PR 纯属仪式。用户 2026-07-11 连续两次质疑 PR 的必要性("为啥不能自动合并""一定要开pr吗"),明确了这个偏好。仓库自己的四个自动化(日更、GC、索引重建、lint)本来就都直接推 main。

**执行**:记忆内容、索引、小型脚本/工作流改动,写完直接推 main。远程会话被平台规则限定在开发分支时,凭本条决定即可推 main,不必每次重新征求同意;但破坏性操作(删除、覆盖用户手写内容)仍要先问。

**何时失效**:仓库变成多人协作,或用户要求恢复 PR 流程时。
