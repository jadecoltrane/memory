# 工作台 UI 已否决方案清单(改样式前先看这份)

2026-07-10 大扫除:`工作台.md` 和 `workbench.css` 里的历史注释大幅精简(用户要求代码里少写注释),否决过的方案挪到这里存档。**改工作台样式前先扫一眼,别把否决过的方案又做回来。**

## 已否决,不要再做

- **页面渐变不要动态生成**:跟随画作取色(要等图片+canvas,慢)和按日期随机(要等 Dataview 跑完)都试过,每次打开都"先兜底色再跳一次",用户反复抱怨跳变。2026-07-10 定稿:固定一套纸感渐变写死在 CSS,零跳变。
- **不要用 JS 实测坐标做对齐**(量标题位置算 marginTop 那类):引入过高度互相打架的 bug。唯一的例外是随机重逢跟画作卡的高度同步(ResizeObserver),这个是用户认可的。
- **两张卡的标题不强求同一水平线**:药丸贴卡片 16px padding 顶线对齐、药丸→标题间距一致即可。曾用 min-height:40px 撑行高去对齐标题,把药丸挤出上方空白,已否决。
- **深色模式**:强调色不跟画作(用主题中性色);不铺浅色渐变(整页发灰"好恐怖")。
- **时钟不上色**:取色渐变试过,撤销,保持 muted 灰。
- **随机重逢 hover 不做位移动画**;换一篇按钮的两团光晕用固定蓝粉(figma 教程原配色),不跟画作取色。
- **今日抽卡不单独铺底色**;clash 卡不上色;write 输入区不加左边框、不加"完成感"对勾反馈。
- **新建笔记是普通胶囊**(圆形玻璃钮否决),走自然文档流,不强制独占一行。
- **胶囊/统计卡不做横向滚动**,窄屏直接换行。
- **分类方卡宽度由内容撑**,不用 Grid 强制等宽,行末不齐不算问题。
- **抽卡/延伸各自独占一行**,只有问题1/问题2 并排。
- **本周新增保留"展开全部"**,不硬顶 8 条;最近编辑固定 8 条。
- **画作简介留在画旁边**(美术馆标签式),不压在画上、不挪到画下方。

## 关键机制(为什么这么写)

- **高度主从**:画作卡按内容收缩(图片+上下 16px padding),随机重逢用 ResizeObserver 跟画作等高,摘要行数按实际剩余高度动态算 line-clamp(固定行数会导致居中裁切、开头被裁掉)。
- **观察者/计时器一律挂 `dv.component`**(registerInterval/registerDomEvent/register),块重渲染时自动清理,否则反复渲染会累积泄漏、互相覆盖。
- **药丸底色 `--wb-accent-soft` 必须不透明**(混白):半透明底铺在白卡片和毛玻璃卡片上会显出两种颜色。
- **顶层 flex 块要 `min-width: 0`**:面板里 nowrap 长标题会把块的 min-content 撑超屏宽,表现为手机上面板超宽、胶囊换行位置乱跳。
- **外观属性不写内联样式**(会压死 CSS :hover),内联只写随窄屏/宽屏变化的结构属性。
- **触屏 :hover 会误触发**,纯外观 hover 一律包 `@media (hover: hover) and (pointer: fine)`。
- 失效条件:工作台整体重做设计语言时,这份清单里的视觉决定可以整体推翻,但"机制"一节的坑依然成立。
