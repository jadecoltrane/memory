// 改成直接测量:先把画作说明文字块的 marginTop 清零,量出它此刻"不加
// 偏移"时的真实顶部坐标,再量"随机重逢"标题此刻的真实顶部坐标,两者的
// 差值就是需要补的偏移量——不管头部多高、标题换不换行、窗口怎么缩放,
// 每次都按当下真实渲染结果重新算,不会像固定像素那样过时。

  const reunionTitleEl = reunionEl.querySelector(".wb-reunion-title");

    captionEl.style.marginTop = "";
    return;
  }
  captionEl.style.marginTop = "0px";
  const delta = reunionTitleEl.getBoundingClientRect().top - captionEl.getBoundingClientRect().top;
  captionEl.style.marginTop = Math.max(0, Math.round(delta)) + "px";
};

  if (paintingEl) new ResizeObserver(syncLayout).observe(paintingEl);
  // 单独再观察随机重逢卡片本身:换一篇按钮会替换标题文字,行数变了
  // 头部高度也会变,不属于"画作卡片尺寸变化"这条路径,得单独盯着
  new ResizeObserver(syncCaptionAlign).observe(reunionEl);
}

> > 单次紧绷感这把尺子测的是瞬时反应,测不出趋势。真正能防慢渗透的是换一把尺子:定期(比如每月)回头问"和这个人/这件事相处一段时间后,我更像自己了,还是更不像自己了",拿的是跨时间的基线对比,而不是单次互动的即时读数——慢性毒素的特征就是它不会让报警器响,只会悄悄挪动基线。[推论,未联网核查]

> [[concepts/控制二分法|控制二分法]]的开放问题里有一条:会不会被滥用成回避的借口——把其实可控(只是难)的事划进"不可控"提前放弃?判据是什么?
>
> > [!write] ✍️ 你的回答
> >我觉得只要不是说放弃了就完全松了一口气，那个就是逃避，只是客观上觉得自己大概率做不到才不做
> >
>
> > [!note]- 💬 我的看法
> > 我倾向于认为这是这套框架最常见的翻车方式,而且判据是清楚的:问自己"如果真的投入努力、技巧或时间,结果发生的概率会不会变"。真的不可控的事(别人的情绪、过去、天气),不管你多努力,概率纹丝不动;假的"不可控"(其实是"太难了"),概率会随投入变化,只是变得慢、变得不舒服。[推论]
> >
> > 还有一个更快的识别信号:说出"这不是我能控制的"之后,如果立刻带来如释重负、然后就此打住、没有任何后续动作——大概率是借口;真正的课题分离说完这句话后,精力会转向"那我能做的部分是什么",而不是停在原地。停顿本身就是破绽。[观点,未联网核查]
