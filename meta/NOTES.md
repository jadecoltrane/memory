# NOTES — notes/ 录入与关联细则

只在两个时机读这份文件:触发词命中、确定要写一条 note 时;整理/梳理 notes 时。平时聊天不用加载。目录定位和触发词在 CLAUDE.md,"该不该进 notes/"的判断在 `meta/RESOLVER.md`——读到这里默认已经确定要写。

## 录入流程

1. 提炼成一条笔记(不是全文粘贴),文件名 = 一句话结论
2. frontmatter 用 `type: note`,加一行 `source:`;可自由建子文件夹分类(懒加载:单篇新主题先放 `notes/` 根目录,同主题第二篇笔记出现时才建子文件夹),索引脚本递归收录
3. 写入仓库、重跑 `python3 meta/scripts/build_memory_index.py`、提交推送到 main

## checkable / verified(先判断是否"可查证")

- **事实性/可查证**(数据、健康、历史、科学结论、他人转述的客观陈述等):
  - 内容来自用户自己亲历/验证过的判断 → `verified: true`;转述自其他 AI、网络、他人、未经交叉核实 → `verified: false`;拿不准时默认 `false`
  - `verified: false` 的会进复核队列,节奏见 `meta/GC.md`
- **主观性/不可查证**(个人想法、感受、散文、创作、经历叙述、审美偏好):frontmatter 加 `checkable: false`,不设 verified 字段,永远不进复核队列——这类内容没有"对不对",原样留存即可

## 图片(截图等)

1. 原图保存到 `notes/assets/YYYY-MM-DD-简短描述.png`(或原格式,不要转码压缩)
2. 对应 note 里用 `![说明](assets/文件名)` 嵌入原图,**并且必须额外写一段文字提炼**——图片本身无法被索引和 grep,未来 AI 靠这段文字才知道这条记忆在讲什么
3. 单张截图正常几百 KB 到几 MB,GitHub 单文件上限 100MB,无需顾虑;但不要把长视频或高清大图仓库当网盘用

## 概念关联与反思(让 notes 活起来)

不只是攒文件,笔记之间要长出联想、张力、甚至矛盾——这是双链图谱存在的意义。

- **每次新写一条 note 时**(不管是响应触发写的,还是发现用户手写、还没建链接的笔记):正文概念出现处挂 `[[concepts/xxx]]`(页面不存在也照写,幽灵节点零成本;写之前先扫 index 的概念列表和已有 aliases,别给同义词开新节点,细则见 `meta/CONCEPTS.md`);再扫一眼 `notes/` 里主题相近的旧笔记(用 meta/index.md 定位候选),找**呼应或冲突**的双链上,双向都加
- **笔记之间不直接互链**,双链优先指向概念页;例外:两篇确实是同一论证的上下篇时可以直链
- **用户手写笔记的整理**(补分类、frontmatter、双链,顺手做张力扫描——笔记间矛盾原样列出提问题,不给结论,不替用户调和;**允许直接改写原文**,拆分、改名、移动):每周 GC 的 4a 步骤或用户说「帮我梳理一下 notes」时做,扫描算法见 `meta/GC.md`
- 找不到有价值的关联或张力时不要硬凑,宁可什么都不说

## GC 边界

notes 只做去重合并,**不自动降级、不自动删除**——删除权只在用户。
