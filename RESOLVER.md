# RESOLVER — 新内容该放哪

按顺序判断,命中第一条就停,不用继续往下看:

1. **用户明确说了**「记下来」「存一下」「收藏这条」「写进记忆库」「remember this」等触发词 → `notes/`(不分流,不管是知识、决定还是感想,一律进这里;若同时构成对 AI 未来行为的约束,`decisions/` 另补一条工作记忆并双链)。完整流程见 CLAUDE.md「我的收藏」一节
2. **用户自己在 Obsidian 手写、推送到了 GitHub,没有触发词** → 也进 `notes/`(GC 或用户主动触发「帮我梳理一下 notes」时补齐分类、frontmatter、双链)
3. 都不是 → 走下面四选一,**满足其一才写,都不满足就什么都不写**:

| 情况 | 写入位置 |
|---|---|
| 做出了一个决定或约定 | `decisions/` |
| 踩了一个坑,且找到了原因和解法 | `pitfalls/` |
| 形成了一个可复用的判断(必须标 confidence) | `insights/` |
| 发现了一个重要但尚未解决的问题 | `questions/` |

**不要写**:代码里 grep 就能查到的事实、一次性任务细节、没有复用价值的流水账。

写完之后:重跑 `python3 scripts/build_memory_index.py` 更新索引;若这条新记忆和已有 2 条以上记忆同属一个主题,同步更新 `SUMMARY.md`(见 CLAUDE.md「结论速览」一节);最后按 CLAUDE.md「写入必须告知」的格式告诉用户。
